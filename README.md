# 🦗 LOCUST - ПОЛНОЕ РУКОВОДСТВО ДЛЯ НАЧИНАЮЩИХ 
 
 
--- 
 
##  ЧТО ТАКОЕ LOCUST? 
 
Locust (в переводе с английского - "саранча") - это инструмент для нагрузочного тестирования. Представьте, что на ваш сайт одновременно заходит 1000 человек. Выдержит ли сайт? Не упадет ли? Не будет ли тормозить? Locust помогает это проверить. 
 
**Почему именно Locust?** 
- Пишем на Python (легко и понятно) 
- Не нужно учить сложные программы с кнопками 
- Видим графики в реальном времени 
- Можно менять нагрузку прямо во время теста 
 
--- 
 
##  ДЛЯ ЧЕГО ЭТОТ ПРОЕКТ? 
 
Этот проект создан, чтобы на практике изучить нагрузочное тестирование. 
 
**Что мы проверяем:** 
1. Как быстро сайт отвечает, когда много пользователей 
2. Есть ли ошибки при нагрузке 
3. Как растет время ответа с увеличением пользователей 
 
--- 
 
##  КАК МЫ ЭТО ДЕЛАЕМ? 
 
Мы создаем виртуальных пользователей (ботов), которые делают то же, что и реальные люди: 
 
**Сценарий поведения пользователя:** 
1. Заходит на сайт и получает список всех постов (это действие частое - вес 7) 
2. Иногда (с вероятностью 30%) нажимает на конкретный пост и читает его (вес 3) 
3. Между действиями "думает" от 1 до 5 секунд (как настоящий читатель)

##  ЧТО МЫ ВИДИМ НА ЭКРАНЕ? 
 
После запуска откроется панель с кучей цифр. Давай разберем, что они значат: 
 
**Основные показатели:** 
 
🔹 **Type** - тип запроса (GET, POST и т.д.) 
🔹 **Name** - какой адрес запрашиваем (/posts или /posts/1) 
🔹 **# Requests** - сколько раз сделали этот запрос 
🔹 **# Fails** - сколько раз была ошибка 
🔹 **Median (ms)** - медианное время ответа (половина запросов быстрее этого времени) 
🔹 **95%%ile (ms)** - время, за которое отвечает 95%% запросов (важнейший показатель!) 
🔹 **Average (ms)** - среднее время ответа 
🔹 **Min (ms)** - минимальное время 
🔹 **Max (ms)** - максимальное время 
🔹 **Current RPS** - запросов в секунду сейчас 
 
--- 
 
##  КАК ПОНИМАТЬ РЕЗУЛЬТАТЫ? 
 
**Хорошие показатели:** 
- Ошибок (Fails) = 0 или очень мало 
- RPS растет с увеличением пользователей 
 
**Плохие показатели:** 
- Много ошибок (сайт падает) 
- 95%%ile  мс (секунда - это долго) 
- Время ответа резко растет при небольшой нагрузке

##  РАЗБОР НАШЕГО КОДА 
 
Посмотрим на файл `src/locustfile.py` и разберем каждую строчку: 
 
```python 
from locust import HttpUser, task, between 
import random 
``` 
**from locust import HttpUser, task, between** - берем из Locust нужные вещи: 
  - HttpUser - это класс одного виртуального пользователя 
  - task - отмечаем, что функция - это задача для пользователя 
  - between - задаем паузу между действиями 
**import random** - нужно для выбора случайного поста 
 
```python 
class BlogUser(HttpUser): 
``` 
Создаем свой класс пользователя, который наследует HttpUser 
 
```python 
    wait_time = between(1, 5) 
``` 
Пользователь будет ждать от 1 до 5 секунд между задачами 
 
```python 
    def on_start(self): 
        response = self.client.get("/posts") 
        if response.status_code == 200: 
            self.posts_list = response.json() 
        else: 
            self.posts_list = [] 
``` 
**on_start** - выполняется, когда пользователь только зашел на сайт 
**self.client.get("/posts")** - делаем запрос к API, получаем список постов 
**response.status_code == 200** - проверяем, что запрос успешный (код 200) 
**self.posts_list = response.json()** - сохраняем список постов в переменную 
 
```python 
    @task(7) 
    def get_all_posts(self): 
        self.client.get("/posts") 
``` 
**@task(7)** - это задача с весом 7 (будет выполняться чаще) 
Функция просто запрашивает список всех постов 
 
```python 
    @task(3) 
    def get_single_post(self): 
        if self.posts_list: 
            post_id = random.choice(self.posts_list)['id'] 
            self.client.get(f"/posts/{post_id}") 
        else: 
            post_id = random.randint(1, 100) 
            self.client.get(f"/posts/{post_id}") 
``` 
**@task(3)** - задача с весом 3 (реже) 
**if self.posts_list:** - если список постов загрузился 
**post_id = random.choice(self.posts_list)['id']** - выбираем случайный пост из списка 
**self.client.get(f"/posts/{post_id}")** - запрашиваем этот пост 
**else:** - если список не загрузился (на всякий случай) 
**post_id = random.randint(1, 100)** - берем случайное число

##  ПРИМЕР ТЕСТА 
 
**Запускаем с параметрами:** 
- 10 пользователей 
- Скорость появления: 1 пользователь в секунду 
 
**Что происходит:** 
- Секунда 1: появился 1 пользователь, делает запросы 
- Секунда 2: появился 2-й пользователь, теперь их 2 
- Секунда 10: появились все 10 пользователей 
 
**Результат через минуту:** 
- Всего запросов: ~300 
- Среднее время: ~60 мс 
- 95%% запросов быстрее 400 мс 
- Ошибок: 0 
 
Вывод: API отлично держит нагрузку 10 пользователей.
