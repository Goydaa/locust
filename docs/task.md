# ТЕХНИЧЕСКОЕ ЗАДАНИЕ 
 
*Полное описание задачи с объяснением для начинающих* 
 
--- 
 
## 1. ОПИСАНИЕ ЗАДАЧИ 
 
**Что мы делаем?** 
 
Мы проводим нагрузочное тестирование публичного API https://jsonplaceholder.typicode.com. 
Это учебный сервис, который отдает вымышленные посты, как маленький блог. 
 
**Зачем?** 
Чтобы научиться: 
- Писать сценарии нагрузки на Python 
- Запускать тесты и видеть результаты 
- Анализировать метрики производительности 
- Понимать, как ведет себя API под нагрузкой 
 
**Что тестируем:** 
- GET /posts - список всех постов (как главная страница блога) 
- GET /posts/{id} - конкретный пост (как страница поста) 
 
--- 
 
## 2. ЦЕЛЬ РАБОТЫ 
 
**Главная цель:** 
Смоделировать поведение 100 реальных пользователей и проверить: 
- Нет ли ошибок (менее 1%%) 
- Как растет время ответа с увеличением нагрузки 
 
**Учебные цели:** 
1. Научиться писать сценарии на Locust 
2. Понять, что такое RPS, latency, процентили 
3. Освоить работу с Git и документацией 
4. Получить практический опыт инженерной работы 
 
--- 
 
## 3. ФУНКЦИОНАЛЬНЫЕ ТРЕБОВАНИЯ 
 
### 3.1. Запуск и управление 
- Система запускается командой: `locust -f locustfile.py --host=https://jsonplaceholder.typicode.com` 
- Открывается веб-интерфейс на http://localhost:8089 
- Можно ввести количество пользователей и скорость появления 
 
### 3.2. Модель пользователя 
- Создать класс `BlogUser`, наследующий `HttpUser` 
- При старте (`on_start`) пользователь получает список всех постов и сохраняет его 
- Между задачами пользователь "думает" от 1 до 5 секунд 
 
### 3.3. Задачи пользователя 
- **Задача 1 (вес 7):** GET /posts - посмотреть список всех постов 
- **Задача 2 (вес 3):** GET /posts/{id} - посмотреть конкретный пост 
  ID выбирается случайно из списка, полученного при старте 
  Если список пуст, брать случайное число 1-100 
 
### 3.4. Сбор статистики 
Locust должен показывать: 
- Количество запросов 
- Количество ошибок 
- Среднее время ответа 
- Медиану и 95-й перцентиль 
- RPS (запросов в секунду)

## 4. НЕФУНКЦИОНАЛЬНЫЕ ТРЕБОВАНИЯ 
 
- **Простота:** Код должен быть понятен новичку 
- **Чистота:** Понятные имена переменных, комментарии 
- **Документация:** Есть README и папка docs с объяснениями 
- **Git:** Минимум 5 коммитов, каждый с осмысленным сообщением 
- **Переносимость:** Работает на Windows, Linux, macOS 
 
--- 
 
## 5. ОГРАНИЧЕНИЯ 
 
- Тестируем только GET-запросы 
- Не используем распределенный режим 
- Не тестируем WebSockets 
- Не сохраняем результаты в БД 
- JSONPlaceholder может ограничивать нагрузку (ошибки 429)
