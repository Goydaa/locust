## 2. С КАКИМИ ТРУДНОСТЯМИ СТОЛКНУЛСЯ 
 
### 2.1. Технические проблемы 
 
**Проблема 1: Установка Python и Git** 
Решение: Переустановил с галочкой "Add to PATH", добавил в переменные среды 
 
**Проблема 2: Кодировка в cmd** 
Решение: Команда `chcp 65001` переключает на UTF-8 
 
**Проблема 3: Ошибка "dubious ownership" в Git** 
Решение: `git config --global --add safe.directory "путь_к_папке"` 
 
**Проблема 4: 404 ошибки в тестах** 
Решение: Добавил загрузку реального списка постов в on_start 
 
### 2.2. Концептуальные сложности 
 
**Сложность 1: Понимание асинхронности** 
Долго не мог понять, почему time.sleep() ломает тест. Оказалось, нужно использовать wait_time. 
 
**Сложность 2: Интерпретация метрик** 
Сначала смотрел только на среднее. Потом понял важность 95-го перцентиля.

## 5. ГДЕ ИНСТРУМЕНТ НЕУМЕСТЕН 
 
❌ **Тестирование фронтенда с JavaScript** - Locust не выполняет JS 
❌ **Простые разовые тесты** - для одного URL проще ab или curl 
❌ **UDP-протоколы** - только HTTP из коробки 
❌ **Максимальная производительность** - Go-инструменты быстрее 
 
--- 
 
## 6. ИТОГОВАЯ ОЦЕНКА 
 
**Оценка: 9/10** 
 
**Плюсы:** 
+ Простота написания тестов на Python 
+ Отличный веб-интерфейс 
+ Гибкость и расширяемость 
+ Активное сообщество 
 
**Минусы:** 
- Сложность с асинхронностью для новичков 
- Документация иногда разрозненна 
- Не подходит для тестирования фронтенда 
 
**Вывод:** Locust - отличный выбор для Python-разработчика, который хочет быстро и эффективно тестировать производительность API.
